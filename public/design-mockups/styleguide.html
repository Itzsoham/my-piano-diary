The one contrast rule

The raw pastels are backgrounds, never text. For text and buttons use the deep tokens (--pink-600/--pink-700, --teal-600/--teal-700). --ink is 12.4:1 on white; --ink-soft is 5.1:1.

--ink-faint (2.7:1) is decorative — uppercase .labels, axis ticks, placeholder text. It must never carry information a user has to read to act. On a coloured fill, only --mint-ink is safe over --grad-mint, and only white is safe over --grad-pink.

03 Status system

Numbers get their own treatment

Anything that is money, a count, or a duration uses --font-num with font-variant-numeric: tabular-nums — see .metric and .money. VND renders as 1.500.000 ₫ (dot grouping, trailing symbol, zero decimals); rate inputs keep their hardcoded đ 250.000 prefix. Supported currencies are VND (default), IDR and USD — nothing else.

06 Layout

Four breakpoints — tablet is the point

This teacher works from an iPad on the piano lid between lessons. Tablet portrait and tablet landscape are the primary targets; desktop is the bonus. The sidebar is the only thing that changes shape.

Phone≤ 639px
Secondary

Off-canvas + bottom tab bar. Sidebar slides in over a scrim from a hamburger. 5 tabs docked at the bottom with --sh-up. One column. Dialogs become sheets.

Tablet portrait640 – 1023
Primary · 768×1024

Off-canvas + tab bar, but the content grid opens to 2 columns. All 44px targets, generous 16–22px gaps. Tables still scroll inside .scroll-x.

Tablet landscape1024 – 1279
Primary · 1024×768

76px icon rail (--rail). Icons only, labels on hover/long-press tooltip. No tab bar — the rail is always reachable with a left thumb. Content gets 2–3 columns.

Desktop≥ 1280px
Bonus

248px full sidebar (--sidebar): brand, “Add lesson” CTA, MAIN + MANAGE groups, user footer. Content maxes at ~1180px and centres.

1
44px is the floor

--tap: 44px. Every button, nav item, chip-as-button, row action and form control clears it — including the 34px .btn--sm, which only ever lives inside a padded row that itself exceeds 44px.

2
Wide content scrolls in its own container

Month grids, report tables and stat strips go inside .scroll-x (plus .no-scrollbar where a scrollbar would be ugly). The page body must never scroll horizontally at any width. If something overflows, it is a bug in that component, not in the shell.

Mon · 4 Tue · 5 Wed · 3 Thu · 6 Fri · 1 Sat · 5 Sun · 2
3
Grid + clamp, never fixed widths

The shell uses grid-template-columns keyed off the sidebar token and minmax() content tracks. Padding is clamp(14px, 3vw, 32px). No width: 1200px anywhere, no horizontal media-query hacks on the page shell.

4
Light theme only

This palette is inherently light; a dark variant would destroy the pastels. Delete the .dark block and the @custom-variant dark line from globals.css rather than maintaining a second theme nobody asked for.

Three things not to get wrong

1. --primary is #c94f7c, not #f3a2be. Bubble Gum is a background; white text on it is 1.96:1 and illegible. The pastel still shows up everywhere — as fills, chips, gradients and the app shell — just never under text.

2. --destructive is pink, not red. A cancelled lesson is a normal fact of a teaching week, not an error state. Keep it in the bubblegum family so the report grid, the chip and the error toast all agree.

3. Charts 2/3/4 are reserved for COMPLETE / PENDING / CANCELLED. If a chart series means a status, it must use the status colour, or the report grid and the bar chart will disagree on the same screen — which is exactly the bug this redesign exists to kill.
