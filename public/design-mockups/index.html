Candy Floss — a redesign of My Piano Diary

Master gallery · 25 mockups, in a resizable device frame

25 mockups · 9 screens · 3 design directions, plus the design system. Dashboard, Calendar and Lessons get five takes each; Reports gets four; Students, Payments, Pieces, Auth and Profile get one. Everything is built tablet-first (768 × 1024 and 1024 × 768 are the primary targets, not an afterthought), light theme only, and drawn from one shared token contract — _theme.css.

Start with Blossom Diary ✿✿ — the new direction, and the one waiting on your verdict. It fuses the other two: The Diary's editorial bones — serif headings, the vertical lesson timeline, generous whitespace — wrapped in Blossom's petals, scalloped edges and Mochi the cat. It's the e-variants: Dashboard, Calendar, Lessons and Reports.

The two it grew out of still run alongside it. Classic is the restrained, information-first take — the a / b / c variants. Sugar Bloom ✿ is the sweet, feminine one — scallops, blossoms, washi tape and Mochi — the d-variants plus the five newer screens. All three share the same palette and the same tokens; only the ornament layer differs, so you can still mix them per-screen and the app reads as one product. Use the filter above the nav to see each direction on its own. styleguide.html carries the Tailwind v4 token handoff for globals.css.

How to use this page
These are static HTML references, not the app. No data is live, nothing saves, and the buttons inside a mockup are for show — every number, name and lesson is realistic sample data (VND, a Da Nang studio). The real product is Next.js + tRPC + Prisma + shadcn with FullCalendar; these files exist to decide the visual direction before any of that is touched.
  • The device switcher above the frame sets the iframe's real width, so the mockup re-renders and its own media queries fire for real — this is not a screenshot being squashed. Widths past the available space are scaled down with a visible scale % so 1440 stays reviewable on a laptop.
  • The direction filter above the nav narrows the list to Classic, Sugar Bloom ✿ or Blossom Diary ✿✿. The design system stays in all three — it is the shared contract.
  • The Design system page carries the token handoff: the palette, the LessonStatus chips and the Tailwind v4 @theme block for globals.css, so a winning mockup ports without re-deriving a single colour.
  • Shortcuts: ↑↓ move between designs · 1–5 switch device · R reloads the frame. Each design is linkable — the URL hash updates as you browse (#dashboard-e/tablet).
