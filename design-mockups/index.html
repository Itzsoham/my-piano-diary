Candy Floss — a redesign of My Piano Diary

Master gallery · every mockup, in a resizable device frame

12 mockups · 4 screens · 3 variants each, plus the design system. Every screen is built tablet-first (768 × 1024 and 1024 × 768 are the primary targets, not an afterthought), light theme only, and drawn from one shared token contract — _theme.css. Same palette, same status chips, same 44px touch targets everywhere, so any combination of winners still looks like one app.

How to review: for each screen, flip through A / B / C at Tablet 768 and Tablet L 1024 — then pick one winner per screen. Mixing (e.g. Dashboard B + Calendar A + Lessons C) is expected and safe; they share the same system.
How to use this page
These are static HTML references, not the app. No data is live, nothing saves, and the buttons inside a mockup are for show — every number, name and lesson is realistic sample data (VND, a Da Nang studio). The real product is Next.js + tRPC + Prisma + shadcn with FullCalendar; these files exist to decide the visual direction before any of that is touched.
  • The device switcher above the frame sets the iframe's real width, so the mockup re-renders and its own media queries fire for real — this is not a screenshot being squashed. Widths past the available space are scaled down with a visible scale % so 1440 stays reviewable on a laptop.
  • The Design system page carries the token handoff: the palette, the LessonStatus chips and the Tailwind v4 @theme block, so a winning mockup ports without re-deriving a single colour.
  • Shortcuts: ↑↓ move between designs · 1–5 switch device · R reloads the frame. Each design is linkable — the URL hash updates as you browse.
